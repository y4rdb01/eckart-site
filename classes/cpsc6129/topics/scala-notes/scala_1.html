Advanced Programming Languages (CPSC 6129)
Scala Programming Language
Introduction

  1. Overview
    • Designed by Martin Odersky at Ecole Polytechnique Federale de Lausanne in 2003
    • Grew out of earlier work on the Pizza programming language by Odersky and others, started in 1997
    • Designed with the goal of leveraging and improving Java (but not to simply "make a better Java")
    • These notes assume familiarity with Java, and will mainly focus on where Scala and Java diverge, instead of rehashing their common comparison operators, logical operators, arithmetic operators, etc.

  2. What is Scala?
    • Scala is a general purpose, statically-typed language with strong type inference
    • Scala is expression-oriented, meaning that most things are expressions that are evaluated rather than statements that are executed
    • Compiles to Java byte code and runs on the Java Virtual Machine (JVM), but can also be interpreted
    • Seamless integration with Java (can call external Java code, use Java objects, extend Java classes)
    • Object-oriented, but has very strong support (and even preference) for functional style of programming
    • Odersky actually objects to object-oriented programming being regarded as a paradigm at odds with others, and instead views object-oriented programming as a complementary approach to the 3 programming paradigms of:
      1. Imperative
      2. Functional
      3. Logic
    • These promoted functional characteristics make mutability easily achievable (i.e. variables which can not be altered once defined and assigned values)
    • This inherent mutability in turn leads to eased concurrency and distributed development

  3. Installing Scala
    • Since Scala runs on the JVM, a Java Runtime Environment (JRE) must already be installed
    • Scala can be downloaded from the Scala website
    • An official build tool, SBT (the Scala Build Tool), provides build functionality similar to Apache ANT or Maven for Scala projects
    • Scala can be installed on Windows, Mac OSX and Linux operating systems:

  4. Scala Basics
    • The Scala intepreted scripting language-style REPL is started by typing the following at a command prompt: 
      
	$ scala
      which presents something similar to the following: 
      
	Welcome to Scala version 2.11.2 (Java HotSpot(TM) 64-Bit Server VM, Java 1.7.0_65).
	Type in expressions to have them evaluated.
	Type :help for more information.

	scala>
    • Scala does away with much of Java's boilerplate, which is what helps make intepretation more feasible
    • A simple Scala Hello World interpreted "program" can be achieved as follows: 
      
	scala> println("Hello, Scala world!")
    • As a compilable program, this is only marginally more verbose: 
      
	object HelloWorld {
	    def main(args: Array[String]) {
	        println("Hello, Scala world!")
	    }
	}
    • With the exception of the object keyword, most of this should be pretty self-explanatory

  5. Scala and Java
    • Scala uses most of the equivalent primitives (and primitive objects) as Java, though they may not be equal (e.g. Scala's Double and Integer are equivalent but not equal to Java's, while Scala actually directly uses the Java String object)
    • This familiarity makes moving from Java to Scala quite easy, as much of Scala's syntax is inherited directly from Java and other related languages (e.g. curly brace conventions, conditional constructs, numerous familiar object-oriented concepts)
    • In fact, Scala's syntax is quite flexible (much more so than Java), and allows for more conventional Java syntax to be used, though it is optional (e.g. semi-colon ending lines, variable type declarations, parenthesis and '.' dot omissions in many circumstances)
    • Scala casing syntax tends to follow snakeCase like Java
    • Scala source code files generally end in the .scala extension, by convention
