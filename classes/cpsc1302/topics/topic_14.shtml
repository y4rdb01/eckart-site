  1. Object Oriented Design (OOD) - the process of planning a system of interacting objects for the purpose of solving a software problem, where objects contain encapsulated data and procedures grouped together to represent entities.
    1. Classes represent a single concept generally associated with nouns, usually things and actors (e.g., Person).
    2. Methods are the actions performed on those nouns (so look for the verbs associated with those concepts).
    3. Properties are internal states of objects that are maintained by class instance variables (sometimes mutiple instance variables are needed to represent a single property - e.g., longitude and lattitude for a location on Earth's surface).
    4. Getters/Setters are methods used to get and set the values of an object's properties - getProperty and setProperty (e.g., getName, setBirthdate).
    5. Property methods return a boolean response to a query (e.g., isStudent, isAdult) and aren't necessarily named for a property.
    6. Non-property based methods usually represent more natural (non-technical) actions associated with the object (e.g., eatLunch, combHair). These methods will still change object property values, but perhaps in less straightforward ways (e.g., eatLunch may increase the weight property - at least until the exercise action is taken).
  2. Sample classes:
    1. Person
    2. Car
    3. University Class (vs Course)
    4. Country (political, geographic)
