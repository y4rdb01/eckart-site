Dr. J Dana Eckart: Computer Science 2 (CPSC 1302) - Throwing Exceptions

  1. Exceptions are created (thrown) when either the Java language system (or your program) recognizes that something has gone wrong.
  2. Exceptions correspond to unusual, uncommon, and generally unexpected situations that can occur during the exeuction of a program. Examples of such situations are:
    1. Division by 0 (zero).
    2. Results of arithmetic operations are too large to store.
    3. Unexpected input error (note that reaching the end of a file is not unusual and unexpected in and of itself).
    4. An indicated/chosen file wasn't found.
    5. An output error occurred (e.g., the disk was full, file was too large).
    6. Part of the program (e.g., a class file) couldn't be located.
    7. The program has run out of memory and can't continue processing.
  3. Exceptions are created by a "throw" statement: 
    
	if (checkingAccount.balance() < 0) {
		throw new Exception("OMG, what happened?!?");
	}
    Note that exceptions are instances of classes (i.e., objects), which can carry information with them about the nature and cause of the error that prompted the exception.
