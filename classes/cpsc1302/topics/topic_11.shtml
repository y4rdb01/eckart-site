  1. A common type of program is one that takes in a lot of data and does one or more of the following actions to the data either printing out the results and/or a summary of the data.
    1. Transformation (e.g., converting units, shifting scales, changing words, formatting the data);
    2. Filtering (e.g., remove temperatures that are too extreme, only show purchases greater than $1, show words that contain letters from a chosen set);
    3. Calculations (e.g., weekly pay based on rate and hours worked, grade points earned for a class); and
    4. Collecting Statistics (e.g., occurence rate of words, invoice total, student GPAs).
  2. Some examples of such programs are:
    1. Creating an invoice based on price and number of each item purchased. Shows sub-total, sales tax, and total amount due.
    2. A student transcript, including the various GPA values.
    3. Class roster.
    4. Statistic calculators (like the one here).
  3. These program broadly share the same typical structure: 
    		 Input
	  	   |
	  	   v
	Computation/Processing
	  	   |
	  	   v
		 Output
  4. In some cases all of the input is read before any/much processing is done, in others, the 3 steps are repeatedly performed on each segment of data (with some final results reported after all of the data segments have been processed. This last version looks like: 
    		 Input <-----------+
	  	   |               |
	  	   v               |
	Computation/Processing     |
	  	   |               |
	  	   v               |
	     Output (optional)     |
	  	   |               |
	  	   +---------------+
	  	   |
	  	   v
		 Output
