Dr. J Dana Eckart: Programming Languages (CPSC 5135) - Symbol Table Implementation

  1. Symbol Tables are used during compile-time to keep track of the various names that are declared (and what they refer to).
  2. Bucket Hashing
  3. Declarations with the same name will be placed into the same bucket.
  4. Because new entries (i.e., named declarations) are placed at the beginning of buckets, they are encountered first.
  5. Thus when a newer (more recently scoped) name hides a previous name, it will appear earlier in the bucket - so its entry will be encountered first when looking for the name.
  6. When a scope is exited, all entries for the current nested scoping level are deleted from the heads of the bucket hash chains.
