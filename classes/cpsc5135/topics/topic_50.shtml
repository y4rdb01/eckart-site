Dr. J Dana Eckart: Programming Languages (CPSC 5135) - Cellular Automata (CA)

  1. Discrete time, discrete space, fixed interaction rules
    1. Cells are represented as a regular N-dimensional tesselation (grid) of "squares" with the edges often wrapped around to the opposing side. Thus a 2-dimensional CA would form a torus (doughnut).
    2. Time progresses in a sequence of unary time steps.
    3. Each cell has a (usually small) set of state variables that represent a finite set of states that the cell can possess.
    4. A (usually small) set of rules maps the current state of each cell, along with the current state of a set of neighboring cells within some fixed and finite (again, usually small) distance, into the next state for this cell.
    5. All cells carry out their rules (which can differ from one another, but normally are the same) in order to calculate what their values will be in the next time step.
    6. Finally, once all cells have completed their calculations they update their states simultaneously before beginning the next time step.
  2. Game of Life
    1. Uses a rectanguilar 2-dimensional grid
    2. Each cell has two states: alive and dead
    3. A cell has 8 neighbors that "touch" the cell at each of the corresponding compass directions: N, S, E, W, NE, SE, SW, NW.
    4. Rules:
      1. Live cells with less than 2 live neighbors die (of loneliness).
      2. Live cells with more than 3 neighbros die (from overcrowding).
      3. Live cells with 2 or 3 live nieghbors remain alive (healthy).
      4. Dead cells with 3 live neighbors become alive (birth).
      5. All other cells retain their previous state.
    5. Example of emergent behavior: Gosper glider gun
    6. Classes of Cellular Automata (when nearly all initial patterns)
      1. Class 1: Quickly settle into a stable homogeneous state. Any randomness in the initial conditions disappears. [Boring]
      2. Class 2: Quickly settle into stable or short-period oscillating structures. Some initial randomness remains but its impact on the final pattern tends to be localized. [Repetitous]
      3. Class 3: Give rise to pseudo-random or chaotic behavior. Any stable structures that emerge are local in both time and space and are quickly destroyed by the surrounding noise (pseudo-randomness). Local changes in the initial pattern tend to propagate globally. [Random]
      4. Class 4: Give rise to interesting and complex behaviors that are neither random nor simple short-period oscillators. Stephen Wolfram has postulated that many (and perhaps all) class 4 CA are Turing Complete. [Self-Organizing]
