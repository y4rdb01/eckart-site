Dr. J Dana Eckart: Programming Languages (CPSC 5135) - Agent-based Modeling (ABM)

  1. Agent-based Modeling
    1. Discrete or Continuous time, discrete space, fixed interaction rules, mobile agents/actor with their own internal state and rules, and optional messages/links for agent communication.
      1. Consist of multiple (individual) actors (agents).
      2. Agents often exist within a modeled physical environment in which they can move around.
      3. Agents can interact directly (via messaging) with one another (though usually only if they are in close proximity).
      4. Agents can interact with their environment (current location) causing a change of state in the local environment, themselves, or both. Note that this enables stigmergic interactions between agents.
      5. Some systems support multiple kinds of agents and allow interaction between individuals, groups, and between individuals and groups.
      6. Unlike Cellular Automata, some ABMs allow the rules governing agent behavior to change - adding an evolutionary or learning aspect to agent actions.
    2. Examples:
      1. SugarScape
      2. Traffic modeling
      3. Economics
  2. CA vs ABM
    1. Both are Turing Complete
    2. Agent-based systems subsume the capabilities of Cellular Automata systems.
    3. It can be difficult to model arbitrary agents of an ABM within a Cellular Automata, so when such capabilites are needed using an ABM is highly recommended.
