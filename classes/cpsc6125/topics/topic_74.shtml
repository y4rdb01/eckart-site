  1. NIST's 3 Cloud service models:
    1. Infrastructure as a Service (IaaS)
      The consumer is able to provision and control processing, storage, networks, and other basic resources in order to deploy and run arbitrary software (e.g., OS, applications), but cannot manage or control the underlying cloud infrastructure.
    2. Platform as a Service (PaaS)
      The consumer can deploy, configure, and control acquired or consumer-created applications built using programming languages, libraries, services, and tools supported by the cloud provider. The consumer cannot manage or control the underlying cloud infrastructure, nor can they provision or control processing, storage, networks, or other basic resources.
    3. Software as a Service (SaaS)
      The consumer is able to use the provider's applications running on a cloud infrastructure, which are accessible from various client devices through either a thin client interface, (e.g., web browser) or program interface. The consumer cannot manage or control the underlying cloud infrastructure (e.g., network, servers, operating systems, storage, or applications) except those limited user specific settings enabled by the application(s).
