Dr. J Dana Eckart: Advanced Operating Systems (CPSC 6125) - Unix Commands

  1. Basic
    1. man - display system documentation
    2. date - write the date and time
    3. cal - print a calendar
    4. bc - An arbitrary precision calculator language
    5. who - display who is on the system
    6. which - locate a command
    7. script - make typescript of terminal session
    8. echo - display a line of text
    9. sleep - suspend execution for an interval
    10. time - time a simple command
    11. Shells are command language interpreters:
      1. sh - the standard (Bourne) shell
      2. tcsh - Enhanced version of the Berkeley C shell (csh) with file name completion and command line editing
      3. ksh - Korn shell enables vi style command line search and editing
      4. bash - Bourne-Again Shell (GNU replacement for sh with elements from csh and ksh)
      5. NOTE: File and directory name wildcards: *, ?
  2. File editing commands
    1. vi, vim - screen-oriented (visual) display editor
    2. sed - stream editor for filtering and transforming text
    3. tr - translate characters
  3. File commands
    1. file - determine file type
    2. cat - concatenate and print files
    3. cp - copy files
    4. mv - move files
    5. more - display files on a page-by-page basis
    6. head - copy the first part of files
    7. tail - copy the last part of a file
    8. sort - sort, merge, or sequence check text files
    9. cut - cut out selected fields of each line of a file
    10. diff - compare two files
    11. wc - word, line, and byte or character count
    12. NOTE: End of file is ctrl-d
  4. File System commands
    1. ls - list directory contents
    2. cd (part of shell)
      1. absolute path names
      2. relative path names
    3. pwd - return working directory name
    4. grep, fgrep, egrep - print lines matching a pattern
    5. find - find files
      If there is a single most powerful command (in terms of its usefulness), this might very well be it. This command has more options than you can shake a finger at. You should spend lots of time reading and rereading this man page and trying out the various options.
    6. mkdir - make directories
    7. rm - remove directory entries
      Be careful using this command with wildcards, as you can easily remove more than you intended, and there's no undelete in Unix.
    8. df - report free disk space
    9. du - estimate file space usage
  5. Process commands
    1. ps - report process status
    2. top - display Linux tasks
    3. kill - terminate or signal processes
      QUOTE: As in certain cults it is possible to kill a process if you know its true name. (Ken Thompson and Dennis M. Ritchie)
    4. history (part of shell)
      1. !# (repeat command # as listed by history)
      2. !! (repeat last command)
      3. !pattern (repeat last command beginning with pattern)
      4. !?pattern (repeat last command containing pattern)
      5. arrow keys (up, down, left, right)
      6. esc?pattern (search for last command containing pattern)
      7. command/filename completion (using tab)
  6. Remote commands
    1. ssh - OpenSSH SSH client (remote login program)
    2. scp - secure copy (remote file copy program)
    3. rsync - a fast, versatile, remote (and local) file-copying tool
